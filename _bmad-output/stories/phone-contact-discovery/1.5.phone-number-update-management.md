# Story 1.5: Phone Number Update Management

## Status

Ready for Review

## Story

**As a** user who changed my phone number,
**I want** to update my phone number in the app,
**so that** contacts can find me with my new number.

## Acceptance Criteria

1. Use existing `setMyPhoneNumber(phoneNumber: String!)` mutation (from Story 1.2)
2. Validate phone format with libphonenumber-js
3. Delete old hash, create new hash (atomic transaction)
4. Preserve existing friend connections
5. Update `User.updatedAt` timestamp
6. Allow no-op if updating to same number
7. Log update events (never log actual numbers)

### Key Implementation Details

**GraphQL Mutation (existing):**
```graphql
mutation SetMyPhoneNumber($phoneNumber: String!) {
  setMyPhoneNumber(phoneNumber: $phoneNumber) {
    id
    hasPhoneNumber
  }
}
```

**Implementation Highlights:**
1. **No-op detection**: Since we use HMAC-SHA256 (deterministic), we can compare hashes directly
2. **Atomic transaction**: Uses Prisma.$transaction() to ensure delete + create + user update are atomic
3. **Audit logging**: Three event types - PHONE_CREATED, PHONE_UPDATED, PHONE_NOOP

**Log Events:**
| Event | Description |
|-------|-------------|
| `PHONE_CREATED` | First time phone registration |
| `PHONE_UPDATED` | Phone number changed to different number |
| `PHONE_NOOP` | Same number submitted (no changes made) |
| `PHONE_INVALID` | Invalid phone format |
| `PHONE_DUPLICATE` | Phone already registered by another user |

**Error Codes:**
| Code | Message | Scenario |
|------|---------|----------|
| `INVALID_PHONE_NUMBER` | "Invalid phone number format" | Cannot parse |
| `PHONE_NUMBER_IN_USE` | "Phone number already registered" | Duplicate |
| `UNAUTHENTICATED` | "Authentication required" | No valid JWT |
