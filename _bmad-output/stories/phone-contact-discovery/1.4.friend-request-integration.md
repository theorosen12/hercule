# Story 1.4: Friend Request Integration

## Status

Approved

## Story

**As a** mobile app user,
**I want** to send friend requests to suggested contacts,
**so that** I can connect with discovered friends seamlessly.

## Acceptance Criteria

1. Verify existing `sendFriendRequest` mutation works with discovered users
2. Add optional `source: "contact_discovery"` parameter for analytics
3. Prevent duplicate friend requests with clear error
4. Handle bidirectional discovery (mutual requests = auto-accept)
5. Document GraphQL workflow for mobile team
6. Return clear errors for blocked/deleted users
7. Trigger push notification on friend request

### Key Implementation Details

**Updated Mutation Signature:**
```graphql
mutation SendFriendRequest($requestedUserId: String!, $source: String) {
  sendFriendRequest(requestedUserId: $requestedUserId, source: $source) {
    id
    status
    createdAt
  }
}
```

**Source Parameter Values:**
| Value | Description |
|-------|-------------|
| `"contact_discovery"` | From getContactSuggestions results |
| `"search"` | From user search |
| `"profile"` | From viewing user profile |
| `null` | Not specified |

**Error Codes:**
| Code | Message | Scenario |
|------|---------|----------|
| `FRIEND_REQUEST_EXISTS` | "Friend request already pending" | Duplicate request |
| `USER_NOT_FOUND` | "User not found" | Target deleted |
| `USER_BLOCKED` | "Cannot send friend request" | Blocked by target |
| `CANNOT_REQUEST_BLOCKED_USER` | "Cannot send friend request" | Target is blocked |
| `ALREADY_FRIENDS` | "Already friends" | Friendship exists |
