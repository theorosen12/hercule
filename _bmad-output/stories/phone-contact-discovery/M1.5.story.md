# Story M1.5: Add Friend Action

## Status: Done

## Story

**As a** user viewing a friend suggestion,
**I want** to add them as a friend directly from the carousel,
**so that** I can connect quickly.

## Acceptance Criteria

1. "Add" button calls existing `sendFriendRequest` mutation with `source: "contact_discovery"`
2. Show loading spinner on button during request
3. On success with `status: "pending"`: card transitions to "En attente" state
4. On success with `status: "accepted"`: card transitions to "Ami ajoutГ©" state (bidirectional auto-accept)
5. Card remains visible with disabled state (not removed)
6. Handle `FRIEND_REQUEST_EXISTS`: show "En attente" state
7. Handle `USER_BLOCKED` / `USER_NOT_FOUND`: remove card silently
8. Handle `CANNOT_REQUEST_BLOCKED_USER`: remove card silently
9. Handle `ALREADY_FRIENDS`: remove card silently
10. Update Apollo cache appropriately

## Technical Notes

- **Do NOT** remove card from carousel on success
- Visual state change: button becomes "En attente" label (disabled)
- Track pending state locally or via Apollo cache

## Implementation

```typescript
// useSendFriendRequest.hook.ts (or use existing)
import { useSendFriendRequestMutation } from '@/shared/infra/types/api.types';

type FriendStatus = 'pending' | 'accepted';

export const useFriendRequestFromDiscovery = () => {
  const [sendRequest, { loading }] = useSendFriendRequestMutation();
  const [statusMap, setStatusMap] = useState<Map<string, FriendStatus>>(new Map());

  const addFriend = async (userId: string) => {
    try {
      const result = await sendRequest({
        variables: {
          requestedUserId: userId,
          source: 'contact_discovery',
        },
      });

      const status = result.data?.sendFriendRequest?.status;

      // Handle auto-accept (bidirectional discovery)
      if (status === 'accepted') {
        setStatusMap((prev) => new Map(prev).set(userId, 'accepted'));
      } else {
        setStatusMap((prev) => new Map(prev).set(userId, 'pending'));
      }

    } catch (error) {
      const errors = extractRequestErrors(error);

      if (errors.FRIEND_REQUEST_EXISTS) {
        // Already pending, just mark it
        setStatusMap((prev) => new Map(prev).set(userId, 'pending'));
      } else if (
        errors.USER_BLOCKED ||
        errors.USER_NOT_FOUND ||
        errors.CANNOT_REQUEST_BLOCKED_USER ||
        errors.ALREADY_FRIENDS
      ) {
        // Remove from suggestions silently
        removeFromCache(userId);
      } else {
        // Show error toast
        showErrorToast(error);
      }
    }
  };

  return { addFriend, loading, statusMap };
};
```

## Apollo Cache Update (for blocked user removal)

```typescript
const removeFromCache = (userId: string) => {
  client.cache.modify({
    fields: {
      getContactSuggestions(existingRefs = [], { readField }) {
        return existingRefs.filter(
          (ref: any) => readField('id', ref) !== userId
        );
      },
    },
  });
};
```

## Error Handling

| Error Code | Behavior |
|------------|----------|
| Success (`status: pending`) | Show "En attente", card stays visible |
| Success (`status: accepted`) | Show "Ami ajoutГ© вң“", card stays visible (auto-accept) |
| `FRIEND_REQUEST_EXISTS` | Show "En attente" (idempotent) |
| `USER_BLOCKED` | Remove card silently from cache |
| `USER_NOT_FOUND` | Remove card silently from cache |
| `CANNOT_REQUEST_BLOCKED_USER` | Remove card silently from cache |
| `ALREADY_FRIENDS` | Remove card silently from cache |
| Network error | Show error toast, button returns to "Add" |
| `RATE_LIMIT_EXCEEDED` | Show rate limit message |

### Auto-Accept Scenario

When User A discovers User B and sends a friend request, if User B has already sent a request to User A, the backend automatically:
1. Accepts both requests
2. Creates the friendship
3. Returns `status: "accepted"` instead of `status: "pending"`

The mobile app should display "Ami ajoutГ©" to indicate immediate friendship.
