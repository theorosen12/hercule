# Story 4.4: Apollo Cache Auto-Update for sendFriendRequest

## Status

Draft

## Story

**As a** user adding friends from the discovery screen,
**I want** the friend request status to update automatically in the UI after I send a request,
**So that** I see the "En attente" state immediately without the app maintaining a separate local state.

## Acceptance Criteria

1. Remove local `requestStatuses` Map state from `FriendDiscoveryContainer`
2. `SuggestionRow` reads friend request status from the `friendRequest` field in the query data (from Apollo cache)
3. After calling `sendFriendRequest` mutation, Apollo cache is automatically updated:
   - The matching `ContactSuggestion` in cache gets its `friendRequest` field set to `{ id, status: 'PENDING' }`
   - The UI re-renders showing "En attente" on that row
4. No manual `requestStatuses` prop drilling through `SuggestionList` → `SuggestionRow`
5. Button states derived from `friendRequest` field:
   - `friendRequest === null` → "Ajouter" button (enabled)
   - Mutation in-flight for this user → Loading spinner
   - `friendRequest.status === 'PENDING'` → "En attente" (disabled)
6. Error handling preserved:
   - `FRIEND_REQUEST_EXISTS` → should now be handled by the cached `friendRequest` status
   - `USER_BLOCKED` / `USER_NOT_FOUND` → remove from list or show error
   - Network errors → allow retry

## Dev Notes

### Apollo Cache Update Strategy

**Preferred approach: `update` function on mutation**

```typescript
const [sendFriendRequest] = useSendFriendRequestMutation();

const handleAddFriend = async (userId: string) => {
  try {
    await sendFriendRequest({
      variables: { userId },
      update: (cache, { data }) => {
        if (!data?.sendFriendRequest) return;

        // Find the ContactSuggestion in the cached query and update its friendRequest
        cache.modify({
          id: cache.identify({ __typename: 'User', id: userId }),
          // Or if ContactSuggestion has its own __typename:
          // id: cache.identify({ __typename: 'ContactSuggestion', id: userId }),
          fields: {
            friendRequest() {
              return {
                __typename: 'FriendRequest',
                id: data.sendFriendRequest.id,
                status: data.sendFriendRequest.status,
              };
            },
          },
        });
      },
    });
  } catch (error) {
    handleMutationError(error, userId);
  }
};
```

**Alternative: Optimistic response**

```typescript
await sendFriendRequest({
  variables: { userId },
  optimisticResponse: {
    sendFriendRequest: {
      __typename: 'FriendRequest',
      id: `temp-${userId}`,
      status: 'PENDING',
    },
  },
  update: (cache, { data }) => {
    // Same cache.modify as above
  },
});
```

This gives instant UI feedback before the server responds.

### Updated SuggestionRow

```typescript
interface SuggestionRowProps {
  suggestion: ContactSuggestion; // includes friendRequest field
  onAddFriend: (userId: string) => void;
  isLoading?: boolean; // for in-flight mutation spinner
}

const SuggestionRow = ({ suggestion, onAddFriend, isLoading }: SuggestionRowProps) => {
  const { friendRequest } = suggestion;

  // Derive button state
  const isPending = friendRequest?.status === 'PENDING';
  const buttonLabel = isPending ? 'En attente' : 'Ajouter';
  const buttonDisabled = isPending || isLoading;

  return (
    <UserLine
      user={suggestion}
      rightElement={
        isLoading ? (
          <ActivityIndicator />
        ) : (
          <Button
            label={buttonLabel}
            onPress={() => onAddFriend(suggestion.id)}
            disabled={buttonDisabled}
          />
        )
      }
    />
  );
};
```

### Updated SuggestionList

```typescript
interface SuggestionListProps {
  suggestions: ContactSuggestion[]; // each includes friendRequest
  onAddFriend: (userId: string) => void;
  loadingUserIds?: Set<string>; // track which rows have in-flight mutations
}

const SuggestionList = ({ suggestions, onAddFriend, loadingUserIds }: SuggestionListProps) => {
  return (
    <FlatList
      data={suggestions}
      renderItem={({ item }) => (
        <SuggestionRow
          suggestion={item}
          onAddFriend={onAddFriend}
          isLoading={loadingUserIds?.has(item.id)}
        />
      )}
      keyExtractor={(item) => item.id}
    />
  );
};
```

### Loading State for Individual Rows

While we remove the `requestStatuses` Map, we still need to track which mutations are in-flight for showing a spinner. This can be a simple `Set<string>` of user IDs currently loading:

```typescript
const [loadingUserIds, setLoadingUserIds] = useState<Set<string>>(new Set());

const handleAddFriend = async (userId: string) => {
  setLoadingUserIds(prev => new Set(prev).add(userId));
  try {
    await sendFriendRequest({ ... });
    // Cache auto-updates, loading clears
  } catch (error) {
    handleMutationError(error, userId);
  } finally {
    setLoadingUserIds(prev => {
      const next = new Set(prev);
      next.delete(userId);
      return next;
    });
  }
};
```

This is much simpler than the previous Map approach because:
- Only tracks "loading" state (not "pending" - that comes from cache)
- Clears after mutation completes (success or failure)
- No need to track "pending" state locally - Apollo cache handles it

### Error Handling Matrix (Updated)

| Error Code | Cache Action | User Feedback |
|------------|-------------|---------------|
| `FRIEND_REQUEST_EXISTS` | Set friendRequest in cache | Row shows "En attente" |
| `USER_BLOCKED` | Evict suggestion from cache | Row disappears |
| `USER_NOT_FOUND` | Evict suggestion from cache | Row disappears |
| Network Error | No cache change | Toast, allow retry |
