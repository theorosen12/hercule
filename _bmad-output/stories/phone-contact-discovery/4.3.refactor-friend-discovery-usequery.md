# Story 4.3: Refactor Friend-Discovery Route with useQuery and Derived State

## Status

Draft

## Story

**As a** developer,
**I want** to simplify the friend-discovery screen by using `useQuery` instead of a lazy query and deriving UI state from the query results,
**So that** the code is simpler, less error-prone, and the UI is always in sync with the data.

## Acceptance Criteria

1. Friend-discovery screen uses `useQuery` (not `useLazyQuery`) for `getContactSuggestions`
2. On mount, the screen extracts contacts from the device (permission is already granted at this point) and passes phone numbers as query variables
3. UI state is derived from query result `{ loading, data, error }`:
   - `loading` → show loading spinner
   - `data.getContactSuggestions.length > 0` → show suggestions list
   - `data.getContactSuggestions.length === 0` → show no-suggestions state
   - `error` → show error state
4. Remove `PhoneEntryBottomSheet` from the friend-discovery screen (phone entry is now on its own route)
5. Remove the state machine (`initial`, `loading`, `suggestions`, `empty`, `denied`) - no longer needed
6. Remove `useContactDiscovery` hook usage in friend-discovery (or simplify it to only extract contacts)
7. Screen still has close (X) button navigating to Home
8. Back button still navigates to Home (no back to phone-number screen)
9. `fetchPolicy: 'network-only'` to ensure fresh suggestions data

## Dev Notes

### Simplified Container

```typescript
const FriendDiscoveryContainer = () => {
  const router = useRouter();

  // Step 1: Extract contacts (permission already granted)
  const { phoneNumbers, deviceContacts, loading: extracting } = useExtractContacts();

  // Step 2: Query suggestions (auto-fires when phoneNumbers ready)
  const { data, loading: querying, error } = useGetContactSuggestionsQuery({
    variables: { phoneNumbers: phoneNumbers ?? [] },
    skip: !phoneNumbers,
    fetchPolicy: 'network-only',
  });

  // Step 3: Derive state
  const isLoading = extracting || querying;
  const suggestions = data?.getContactSuggestions ?? [];
  const hasSuggestions = suggestions.length > 0;

  const handleClose = () => router.replace('/');

  return (
    <SafeAreaView>
      <Header>
        <Title>Tes amis sont sur Nara !</Title>
        <CloseButton onPress={handleClose} />
      </Header>
      <Content>
        {isLoading && <LoadingSpinner />}
        {!isLoading && hasSuggestions && (
          <SuggestionList suggestions={suggestions} />
        )}
        {!isLoading && !hasSuggestions && !error && (
          <NoSuggestionsState deviceContacts={deviceContacts} />
        )}
        {error && <ErrorState />}
      </Content>
    </SafeAreaView>
  );
};
```

### useExtractContacts Hook

```typescript
const useExtractContacts = () => {
  const [phoneNumbers, setPhoneNumbers] = useState<string[] | null>(null);
  const [deviceContacts, setDeviceContacts] = useState<DeviceContact[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const extract = async () => {
      // Permission is already granted (checked on phone-number screen)
      const { data } = await Contacts.getContactsAsync({
        fields: [Contacts.Fields.PhoneNumbers, Contacts.Fields.Name],
      });

      const numbers = extractAndNormalizePhoneNumbers(data); // reuse existing util
      const contacts = extractDeviceContacts(data); // reuse existing util

      setPhoneNumbers(numbers);
      setDeviceContacts(contacts);
      setLoading(false);
    };

    extract();
  }, []);

  return { phoneNumbers, deviceContacts, loading };
};
```

### State Derivation (Before vs After)

| Before (State Machine) | After (Derived) |
|-------------------------|-----------------|
| `state === 'initial'` | N/A (no longer needed) |
| `state === 'loading'` | `extracting \|\| querying` |
| `state === 'suggestions'` | `!loading && suggestions.length > 0` |
| `state === 'empty'` | `!loading && suggestions.length === 0` |
| `state === 'denied'` | N/A (can't reach this screen without permission) |

### Why No Permission Denied State?

The phone-number screen (Story 4.1) only navigates to friend-discovery if permission is granted. If permission is denied, the user goes to Home. Therefore, the friend-discovery screen can assume permission is always granted.
