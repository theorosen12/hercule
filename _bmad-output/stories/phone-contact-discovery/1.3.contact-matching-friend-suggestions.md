# Story 1.3: Contact Matching and Friend Suggestions Query

## Status

Ready for Review

## Story

**As a** mobile app user,
**I want** to see which of my contacts are on the platform,
**so that** I can discover and connect with friends.

## Acceptance Criteria

1. Create `getContactSuggestions(phoneNumbers: [String!]!): [User!]!` query accepting contacts as parameter
2. Normalize and hash provided phone numbers at runtime (no storage)
3. Match hashed contacts against `HashedPhoneNumber` table (registered users' phones)
4. Return matched users with fields: id, username, firstName, lastName, profilePictureMediaId
5. Sort results alphabetically by firstName, lastName
6. Exclude requesting user from results
7. Exclude existing friends
8. Exclude users with pending friend requests (both directions)
9. Return empty array (not null) when no matches
10. Reject requests >1,000 contacts with clear error
11. Complete query within 2 seconds for 1,000 contacts (includes hashing time)
12. Optional: Brief cache (5 min) based on hash of input contacts

## Dev Notes

### Important Implementation Change

**Changed hashing from bcrypt to HMAC-SHA256:**
- bcrypt produces different hashes for the same input (random salt)
- This makes it impossible to do `WHERE hashedPhone IN (...)` lookups
- HMAC-SHA256 is deterministic and allows efficient database matching
- Security is maintained via server-side secret key (`PHONE_HASH_SECRET` env var)
- Performance improved: ~19ms for 1000 phone numbers vs potential minutes with bcrypt
