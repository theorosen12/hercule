# Story 2.4: Add Friend Action in Signup Flow

## Status

Complete

## Story

**As a** new user viewing friend suggestions on the Friend Discovery page,
**I want** to add friends directly from the signup flow,
**So that** I can connect with them immediately.

## Acceptance Criteria

1. "Ajouter" button on each `SuggestionRow` calls `sendFriendRequest` mutation
2. Include `source: "signup_contact_discovery"` for analytics differentiation
3. On success: row transitions to "En attente" state
4. Row remains visible with disabled state
5. User can add multiple friends before continuing
6. Handle errors gracefully (already friends, blocked user, etc.)
7. Update Apollo cache appropriately

### State Management in FriendDiscovery Container

```typescript
// Local state for tracking pending requests
const [requestStatuses, setRequestStatuses] = useState<Map<string, 'pending' | 'loading'>>(new Map());

const handleAddFriend = async (userId: string) => {
  setRequestStatuses(prev => new Map(prev).set(userId, 'loading'));

  try {
    await sendFriendRequest({
      variables: { userId, source: 'signup_contact_discovery' }
    });
    setRequestStatuses(prev => new Map(prev).set(userId, 'pending'));
  } catch (error) {
    setRequestStatuses(prev => {
      const next = new Map(prev);
      next.delete(userId);
      return next;
    });
    // Handle error
  }
};

// In render
<SuggestionList
  suggestions={suggestions}
  onAddFriend={handleAddFriend}
  requestStatuses={requestStatuses}
/>
```

### Row States

| State | Button | Appearance |
|-------|--------|------------|
| Default | "Ajouter" (enabled) | Normal styling |
| Loading | Spinner | Button disabled |
| Pending | "En attente" (disabled) | Muted styling |
| Error | "Ajouter" (enabled) | Toast shown |

### Error Handling Matrix

| Error Code | Action | User Feedback |
|------------|--------|---------------|
| `FRIEND_REQUEST_EXISTS` | Set pending state | Show "En attente" |
| `USER_BLOCKED` | Remove row | None (silent) |
| `USER_NOT_FOUND` | Remove row | None (silent) |
| Network Error | Allow retry | Toast message |
| Unknown Error | Log, allow retry | Toast message |
