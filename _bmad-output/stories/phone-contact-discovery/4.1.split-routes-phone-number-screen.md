# Story 4.1: Split Routes - Create Phone-Number Screen with Permission & Navigation

## Status

Draft

## Story

**As a** newly registered user,
**I want** a dedicated phone number entry screen as the first step after registration,
**So that** the flow is clearer, each screen has a single responsibility, and navigation to friend-discovery only happens when we have permission and a phone number.

## Acceptance Criteria

1. Create new route at `/register/phone-number` (authenticated route)
2. Screen displays phone number entry as a full-screen form (not a BottomSheet)
3. Reuse `PhoneNumberCard` / `PhoneNumberCardContainer` component from Epic 1
4. Screen has a skip/close (X) button to navigate directly to Home
5. On phone number submit (form validation + mutation success):
   - Request contact permission via `expo-contacts` (system dialog)
   - If permission **granted** + phone saved → `router.replace('/register/friend-discovery')`
   - If permission **denied** + phone saved → `router.replace('/')` (Home)
6. If user skips phone entry (close/X button) → `router.replace('/')` (Home)
7. Update `handlePostRegistration` to navigate to `/register/phone-number` instead of `/register/friend-discovery`
8. Back button/gesture navigates to Home (cannot go back to registration)

### Container Structure

```typescript
const PhoneNumberScreenContainer = () => {
  const router = useRouter();
  const { user } = useCurrentUserContext();

  // If user already has a phone number, skip to friend-discovery or home
  useEffect(() => {
    if (user.hasPhoneNumber) {
      // User already has phone, go request permission and decide
      handleExistingPhone();
    }
  }, []);

  const handlePhoneSuccess = async () => {
    // 1. Ask contact permission
    const { status } = await Contacts.requestPermissionsAsync();

    if (status === 'granted') {
      // 2. Navigate to friend discovery
      router.replace('/register/friend-discovery');
    } else {
      // 3. Navigate to home
      router.replace('/');
    }
  };

  const handleClose = () => {
    router.replace('/');
  };

  return (
    <SafeAreaView>
      <Header>
        <Title>Retrouve tes amis</Title>
        <CloseButton onPress={handleClose} />
      </Header>
      <Content>
        <PhoneNumberCardContainer
          onSuccess={handlePhoneSuccess}
        />
      </Content>
    </SafeAreaView>
  );
};
```

### Navigation Flow

```
handlePostRegistration()
    |
    v
/register/phone-number
    |
    +---> [Submit phone]
    |         |
    |         +---> Permission granted → /register/friend-discovery
    |         +---> Permission denied  → / (Home)
    |
    +---> [Close/Skip] → / (Home)
```

### Edge Case: User Already Has Phone Number

If the user somehow already has a phone number (e.g., set previously), the screen should:
1. Request contact permission directly
2. Navigate accordingly (friend-discovery or home)

This handles the case where a user returns to this flow.
