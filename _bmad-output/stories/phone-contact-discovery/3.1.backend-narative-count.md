# Story 3.1: Backend - Add Narative Count to Contact Suggestions

## Status

Draft

## Story

**As a** mobile app,
**I want** the `getContactSuggestions` query to return narative membership count for each user,
**So that** I can display how active each suggested friend is on the platform.

## Acceptance Criteria

1. Modify `getContactSuggestions` query response to include `narativeCount: Int!` field
2. Count ALL naratives where the user is a member (not just public)
3. Optimize query to avoid N+1 problem (use aggregation or dataloader)
4. Return 0 if user is not member of any narative
5. Performance: query should remain under 2 seconds for 1000 contacts

### Implementation Approach (Repository Pattern)

**Step 1: Repository Method**
```typescript
// narative-member.repository.ts (or contact-discovery.repository.ts)

async getNarativeCountsByUserIds(userIds: string[]): Promise<Map<string, number>> {
  if (userIds.length === 0) {
    return new Map();
  }

  const counts = await this.prisma.narativeMember.groupBy({
    by: ['userId'],
    where: { userId: { in: userIds } },
    _count: { narativeId: true },
  });

  const countMap = new Map<string, number>();
  counts.forEach((c) => {
    countMap.set(c.userId, c._count.narativeId);
  });

  return countMap;
}
```

**Step 2: Service Layer**
```typescript
// contact-discovery.service.ts

async getContactSuggestions(phoneNumbers: string[], currentUserId: string): Promise<ContactSuggestion[]> {
  // ... existing logic to get matched users ...
  const matchedUsers = await this.userRepository.findByHashedPhones(hashedPhones);

  // Get narative counts from repository
  const userIds = matchedUsers.map(u => u.id);
  const narativeCounts = await this.narativeMemberRepository.getNarativeCountsByUserIds(userIds);

  // Merge data
  return matchedUsers.map(user => ({
    id: user.id,
    firstName: user.firstName,
    lastName: user.lastName,
    username: user.username,
    profilePictureMediaId: user.profilePictureMediaId,
    narativeCount: narativeCounts.get(user.id) ?? 0,
  }));
}
```

**Step 3: Resolver (unchanged)**
```typescript
// contact-discovery.resolver.ts
@Query(() => [ContactSuggestion])
async getContactSuggestions(
  @Args('phoneNumbers', { type: () => [String] }) phoneNumbers: string[],
  @CurrentUser() user: User,
): Promise<ContactSuggestion[]> {
  return this.contactDiscoveryService.getContactSuggestions(phoneNumbers, user.id);
}
```

### Database Schema Reference

```prisma
model NarativeMember {

  // ...

```

### Performance Considerations

- Use `groupBy` aggregation instead of counting per user
- DataLoader batches multiple requests into single query
- Consider Redis cache for very active users (optional)
- Index on `NarativeMember.userId` should already exist
